# 장애 대응 매뉴얼

> 최종 수정일: YYYY-MM-DD

## 📌 개요

장애 발생 시 대응 절차를 정의합니다.

## 🚨 장애 등급

| 등급 | 정의 | 대응 시간 |
|------|------|----------|
| P0 | 서비스 전면 중단 | 즉시 |
| P1 | 핵심 기능 장애 | 30분 이내 |
| P2 | 일부 기능 오류 | 2시간 이내 |
| P3 | 경미한 버그 | 24시간 이내 |

## 📞 비상 연락망

| 순위 | 역할 | 이름 | 연락처 |
|------|------|------|--------|
| 1 | 개발 리드 | | |
| 2 | 인프라 담당 | | |
| 3 | PO | | |

## 🔥 장애 대응 프로세스

### 1. 인지 (Detection)
- 모니터링 알림 확인
- 사용자 신고 접수

### 2. 분류 (Triage)
- 장애 등급 판단
- 담당자 호출

### 3. 대응 (Response)
- 원인 파악
- 긴급 조치 (롤백 등)

### 4. 복구 (Recovery)
- 정상화 확인
- 서비스 재개

### 5. 사후 분석 (Post-mortem)
- 원인 분석
- 재발 방지 대책

## 📋 주요 장애 시나리오

### 서버 다운
1. Cloud Console에서 Cloud Run 상태 확인
2. 로그 확인: `gcloud logging read ...`
3. 필요시 재배포 또는 롤백

### DB 연결 실패
1. Cloud SQL 상태 확인
2. 연결 정보 확인
3. 필요시 인스턴스 재시작

### 트래픽 급증
1. 오토스케일링 확인
2. 필요시 수동 스케일업
3. Rate Limiting 적용
