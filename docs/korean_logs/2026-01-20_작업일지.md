## 🛠️ 기술적 해결 (Engineering Execution)

### 1. Vercel 배포 메커니즘 정규화 (Vercel Normalization)
모노레포의 복잡한 경로 문제를 정석적인 방법으로 해결했습니다.
- **Root Directory 최적화**: `apps/web`을 루트로 정규화하고, 누적 경로(Duplicate paths) 에러를 방지하기 위해 **Output Directory**를 기본값(`.next`)으로 리셋했습니다.
- **Config Conflict 해결**: 루트의 `vercel.json`을 삭제하여 Vercel UI 설정과 파일 설정 간의 충돌을 원천 차단했습니다.

### 2. CI/CD 파이프라인 정합성 확보 (Build Stability)
다양한 버전 충돌과 ESM 환경의 제약을 극복했습니다.
- **Version Alignment**: 프로젝트 사양에 맞춰 Node.js 버전을 **v20 (LTS)**으로, PNPM을 **v9.15.2**로 강제 통일하여 환경 불일치로 인한 런타임 에러를 방지했습니다.
- **ESLint Config Migration**: ESM 환경에서 발생한 JSON import 에러(`ERR_IMPORT_ATTRIBUTE_MISSING`)를 해결하기 위해 `.eslintrc.json`을 `.eslintrc.cjs`로 변환하여 하위 호환성을 확보했습니다.

### 3. 법규 및 마케팅 자산 구축
- **Compliance Implementation**: 설명회 전 법무 리스크 방지를 위해 `/terms`(이용약관) 및 `/privacy`(개인정보처리방침) 페이지를 긴급 구현했습니다.
- **Education Landing**: 1기 설명회를 위한 랜딩 페이지(`/education/session`)를 구축하여 비즈니스 준비를 마쳤습니다.
