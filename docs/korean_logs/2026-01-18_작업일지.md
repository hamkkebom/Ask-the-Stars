# 📅 2026-01-18 작업 일지: UX/DX 동기화

## 💡 작업 배경 (Context)
관리자 대시보드에서 '정산 내역 확인'과 '프로젝트 캘린더'는 빈번하게 사용되는 기능입니다. 그러나 기존에는 단순 탭 전환을 위해 페이지 전체를 이동(새로고침)해야 했으며, 이는 사용자 경험(UX)을 끊기게 만들고 서버 리소스를 낭비하는 원인이었습니다.

---

## 🛠️ 기술적 해결 (Engineering Execution)

### 1. Query Parameter 기반 SPA 패턴 도입
'정산 관리' 페이지를 단일 엔트리 포인트(`earnings/page.tsx`)로 통합하고, `useSearchParams`를 활용해 뷰를 제어했습니다.
- **기술**: Next.js Server Component + Client Component 하이브리드 패턴.
- **구현**: `?type=PRIMARY` 및 `?type=SECONDARY` 쿼리에 따라 데이터를 실시간으로 필터링.
- **삭제**: 중복 코드를 유발하던 `earnings/primary`, `earnings/secondary` 물리적 폴더 제거.

### 2. 조건부 렌더링 아키텍처 (Calendar View)
'내 프로젝트' 페이지에서 리스트 뷰와 캘린더 뷰를 `?view=calendar` 파라미터 하나로 즉시 전환할 수 있게 만들었습니다.
- **성능**: 뷰 전환 시 페이지 리로드 없이 클라이언트 상태만 변경되어 즉각적인 반응 속도 확보.

---

## 🚀 비즈니스 임팩트 (Business Value)
- **사용자 경험(UX)**: 깜빡임 없는(Seamless) 화면 전환으로 앱과 같은 부드러운 사용성을 제공합니다.
- **개발자 경험(DX)**: 유지보수해야 할 파일 수가 줄어들고, 라우팅 로직이 한곳으로 응집되어 코드가 간결해졌습니다.
