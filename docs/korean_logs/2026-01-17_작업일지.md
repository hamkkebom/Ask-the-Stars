# 📅 2026-01-17 작업 일지: 디자인 시스템 고도화

## 💡 작업 배경 (Context)
초기 MVP 모델에서는 모든 페이지가 동일한 `Hero` 섹션을 공유하고 있어, 각 페이지(비전, 연혁, 문화 등)의 고유한 아이덴티티를 표현하기 어려웠습니다. 브랜드 스토리텔링 강화를 위해 **페이지별 커스텀 디자인이 가능한 구조**로 확장이 필요했습니다.

---

## 🛠️ 기술적 해결 (Engineering Execution)

### 1. Hero 컴포넌트 탈부착 아키텍처 구현
기존 `layout.tsx`에 하드코딩되어 있던 Hero 섹션을 제거하고, 각 페이지 레벨로 제어권을 이양했습니다.
- **Before**: `Layout`이 Hero를 소유 → 모든 하위 페이지 디자인 강제 통일.
- **After**: `Layout`은 `children`만 렌더링하고, 각 `Page`가 자신만의 `Hero` 컴포넌트를 소유.
- **영향 범위**: `apps/web/src/app/(public)/about/*`

### 2. 코드 품질 표준화 (Linting)
팀 규모 확장에 대비해 일관된 문서 작성 규칙을 정의했습니다.
- **설정**: `.markdownlint.json` 도입.
- **정책**: 가독성을 해치는 과도한 규칙(MQ001, MQ025)은 완화하고, 필수 포맷팅 규칙은 강화.

---

## 🚀 비즈니스 임팩트 (Business Value)
- **브랜딩 강화**: 각 소개 페이지(`History`, `Vision`)가 100% 독자적인 톤앤매너를 가질 수 있게 되었습니다.
- **협업 효율**: 명확한 린트 규칙 덕분에 PR 리뷰 시 스타일 논쟁(Nitpicking)이 줄어들고 로직 리뷰에 집중할 수 있습니다.
