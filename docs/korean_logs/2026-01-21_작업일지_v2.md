## 🛠️ 세부 작업 내용

### 1. 신청 신청 내부화 (Lead Funnel Integration)
- **DB 스키마**: [schema.prisma](file:///packages/database/prisma/schema.prisma)에 `Lead` 모델 추가. 신청자 정보(이름, 이메일, 연락처, 관심분야 등)를 Supabase에서 직접 관리.
- **백엔드**: `leads` 모듈 생성. `POST /leads` 엔드포인트를 통해 데이터 검증 및 저장 로직 구현.
- **프론트엔드**: 기존 구글 폼 링크를 대체하는 고품질 [LeadForm](file:///apps/web/src/components/features/leads/lead-form.tsx) 컴포넌트 개발. Framer Motion을 활용한 애니메이션 및 성공/에러 상태 처리 적용.

### 2. 실시간 알림 인프라 구축
- **게이트웨이**: [NotificationsGateway](file:///apps/api/src/modules/notifications/notifications.gateway.ts) 구현. Redis 어댑터를 통해 클러스터링 환경에서도 실시간 알림 발송 가능 확인.
- **기능**: 설명회 도중 실시간 공지 및 사용자 상태 체크를 위한 소켓 이벤트 정의.

### 3. UI/UX 폴리싱 및 빌드 최적화
- **Tailwind**: 새로운 명세에 맞춰 그라디언트 클래스(`bg-linear-to-*`) 정합성 확보.
- **빌드 안정화**: 백엔드 빌드 시 발생하던 TS Strict 모드 관련 에러(Property Initialization 등) 해결.
