# Security Policy

## 지원 버전

| 버전 | 지원 상태 |
|------|-----------|
| 최신 | ✅ 지원 |
| 이전 | ❌ 미지원 |

## 취약점 신고

보안 취약점을 발견하셨다면 **공개 이슈로 등록하지 마시고** 아래 방법으로 비공개 신고해 주세요:

1. **이메일**: [보안 담당자 이메일]
2. **GitHub Security Advisories**: 저장소의 Security 탭 → "Report a vulnerability"

### 신고 시 포함할 정보

- 취약점 유형
- 영향을 받는 파일/코드 경로
- 재현 단계
- 잠재적 영향 범위

### 응답 기한

- **확인**: 영업일 기준 48시간 이내
- **초기 평가**: 7일 이내
- **수정 완료**: 심각도에 따라 다름

## 보안 모범 사례

이 프로젝트에서 준수하는 보안 원칙:

- ✅ 환경 변수로 민감 정보 관리
- ✅ `.env` 파일 Git 추적 제외
- ✅ HTTPS 통신 필수
- ✅ 의존성 정기 업데이트
